# FastWhisper — передача проекта для продолжения на компьютере

Это инструкция для продолжения работы в Claude Code на Windows-машине.
Написана после разработки в облачном Linux-контейнере, где Windows-часть
физически нельзя было проверить. Здесь честно перечислено, что проверено,
что нет, и в чём я сам сомневаюсь.

## 1. Как забрать код

Лучший способ — через git (весь код уже запушен):

```powershell
git clone https://github.com/thecruelbaes/redroom-new.git
cd redroom-new
git checkout claude/fastwhisper-windows-dictation-5uc208
cd fastwhisper
```

Альтернатива — распаковать приложенный `fastwhisper-project.zip` в любую
папку. Содержимое идентично ветке.

Дальше:

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m unittest discover -s tests   # должно быть: Ran 32 tests ... OK
```

В папке уже лежит `CLAUDE.md` — открой проект в Claude Code, и контекст
подхватится автоматически. Первая команда для Claude Code на компьютере:
«Прогони чек-лист из HANDOFF.md раздел 3 и почини всё, что упадёт».

## 3. Чек-лист первого запуска (по порядку, чинить по мере падения)

1. `python -m fastwhisper --file testdata\jfk_en.wav` — скачается модель
   (small на CPU / large-v3-turbo на GPU), напечатает фразу Кеннеди.
2. `python -m fastwhisper --file testdata\silence_5s.wav` — пустой вывод,
   «No speech detected», модель даже не загружается.
3. `python -m fastwhisper` — дождаться серой иконки в трее; в Блокноте
   зажать Ctrl+Space, сказать фразу по-русски, отпустить — текст вставился.
4. То же по-английски (автоопределение языка).
5. Скопировать что-то в буфер, продиктовать, проверить, что буфер
   восстановился.
6. Диктовка >30 секунд — в логе `chunks=2+`, текст склеен нормально.
7. Строка `Latency: ...` в логе — сравнить с целями (<2с GPU / <5с CPU
   для 10-секундной фразы).
8. Облако: `cloud.enabled: true` в config.yaml, `$env:POLZA_API_KEY`,
   повторить п. 1 и 3.

## 4. В чём я сомневаюсь (проверять в первую очередь)

Это места, где код написан «вслепую» без возможности запуска — вероятность
ошибок здесь выше всего:

1. **SendInput через ctypes (insert.py)** — структуры `_KEYBDINPUT`/`_INPUT`
   собраны вручную под x64. Если раскладка структур неверна, паста молча не
   сработает или уронит процесс. Есть фолбэк на посимвольный ввод, но сам
   путь надо проверить первым. Если сломано — проще всего проверить ровно
   этот модуль изолированно: `python -c "from fastwhisper.insert import _send_ctrl_v; _send_ctrl_v()"`.
2. **Ctrl+Space как хоткей по умолчанию** — на Windows он может конфликтовать
   с переключателем IME/раскладки и с шорткатами приложений (например,
   автодополнение в IDE). Если мешает — поменять `hotkey` в config.yaml
   (например, на `ctrl+alt+space` или `f9`); код это уже умеет.
3. **Вставка при ещё зажатом физическом Ctrl** — если транскрипция очень
   быстрая, Ctrl+V уйдёт, пока пользователь держит Ctrl; должно быть
   безвредно, но не проверено. Также классическая консоль (conhost) может
   игнорировать Ctrl+V — там спасает `insertion.method: type`.
4. **Постоянно открытый микрофонный поток (capture.py)** — сделан ради
   мгновенного старта записи, но Windows будет показывать индикатор «микрофон
   используется» всё время работы приложения. Если это неприемлемо — придётся
   открывать поток по нажатию (цена: ~100-300 мс обрезки начала фразы; тогда
   добавить преролл-буфер).
5. **pystray + keyboard + рабочие потоки** — трей в главном потоке, хуки
   клавиатуры и транскрипция в своих. На Windows это типовая схема, но
   взаимодействие именно этих трёх библиотек вживую не проверялось.
6. **base_url облака** (`https://api.polza.ai/v1`) — я его НЕ проверял,
   это предположение по аналогии с OpenAI. Сверить с документацией polza.ai:
   и путь (`/audio/transcriptions`), и формат ответа (ожидается `{"text": ...}`),
   и имя модели (`whisper-1`).
7. **Пороги эвристик** — фильтр галлюцинаций (no_speech_prob > 0.6 и
   avg_logprob < -1.0), список фраз-артефактов, VAD-пороги (silence 300 мс)
   взяты из практики сообщества, а не подобраны на ваших данных. После живых
   прогонов их надо тюнить. Русского тестового WAV в testdata нет — стоит
   записать и добавить.
8. **cuDNN 9 / cuBLAS для GPU** — самое частое место падения faster-whisper
   на Windows. Если модель не грузится на GPU, код откатится на CPU+small,
   но это молча съест качество: смотреть лог `Loading model ...`.
9. **`keyboard.write` для кириллицы** (режим `type` и фолбэк) — должен
   работать через юникодные события, но не проверялся.

## 6. Карта кода

```
fastwhisper/
  __main__.py   CLI: --file / live-режим, автоподхват config.yaml
  config.py     dataclass-конфиг + config.yaml, ключ облака из env
  vad.py        Silero VAD (ONNX из faster-whisper), 16 кГц
  chunking.py   чистая логика нарезки по паузам (юнит-тесты)
  postprocess.py фильтр галлюцинаций + склейка текста (юнит-тесты)
  asr.py        LocalASR (faster-whisper, GPU→CPU фолбэк), build_asr
  cloud.py      CloudASR (urllib, multipart WAV, CloudError)
  pipeline.py   VAD → чанки → ASR → фильтр, тайминги
  app.py        живой режим: трей + хоткей + запись + вставка + латентность
  capture.py    микрофон (sounddevice), постоянный поток
  hotkey.py     push-to-talk (keyboard)
  insert.py     буфер + Ctrl+V (SendInput/ctypes), восстановление буфера
  tray.py       pystray, 4 статуса
tests/          32 теста (unittest, без pytest — намеренно)
testdata/       jfk_en.wav (речь, 11 с), silence_5s.wav (тишина)
config.yaml     все настройки с комментариями
README.md       инструкция пользователя (по-русски)
```
