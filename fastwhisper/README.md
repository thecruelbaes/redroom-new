# FastWhisper

Локальная диктовка для Windows в стиле Wispr Flow / WhisperMe: зажали глобальную
горячую клавишу — говорите — отпустили — распознанный текст вставляется в
активное окно. Русский и английский, автоопределение языка. По умолчанию всё
работает полностью офлайн на faster-whisper (CTranslate2).

## Требования

- Windows 10/11, Python 3.11+
- Микрофон
- Опционально: NVIDIA GPU с CUDA 12 + cuDNN 9 — тогда используется модель
  `large-v3-turbo` (int8). Без GPU автоматически берётся модель `small` на CPU.

## Установка

```powershell
cd fastwhisper
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

При первом запуске модель скачивается с Hugging Face (~1.6 ГБ для
`large-v3-turbo`, ~500 МБ для `small`) и кэшируется в
`%USERPROFILE%\.cache\huggingface`.

Примечание про CUDA: faster-whisper использует CTranslate2, которому нужны
библиотеки cuBLAS и cuDNN 9 для CUDA 12. Проще всего поставить их через pip:
`pip install nvidia-cublas-cu12 nvidia-cudnn-cu12` — либо установить CUDA
Toolkit + cuDNN системно. Если CUDA недоступна или загрузка модели на GPU
падает, приложение само откатывается на CPU + `small`.

## Запуск

### Живой режим (диктовка)

```powershell
python -m fastwhisper
```

В трее появится иконка:

- синяя — модель загружается (горячая клавиша ещё не активна);
- серая — готов к работе;
- красная — идёт запись (клавиша зажата);
- жёлтая — распознавание.

Зажмите **Ctrl+Space**, говорите, отпустите — текст вставится в активное окно.
Выход — через пункт «Exit» в меню иконки трея.

Запускать терминал лучше от администратора: глобальные хуки клавиатуры
(библиотека `keyboard`) в ряде случаев требуют повышенных прав.

### CLI-режим (проверка на файле)

```powershell
python -m fastwhisper --file testdata\jfk_en.wav
python -m fastwhisper --file test.wav --model tiny --language ru -v
```

Прогоняет полный пайплайн (VAD → ASR → фильтр галлюцинаций) и печатает
результат. `--model` подменяет модель (удобно для быстрых проверок),
`-v` включает отладочные логи с таймингами.

## Конфигурация

Все настройки — в `config.yaml` (см. комментарии в файле). Основное:

- `model` — имя модели, фолбэк, устройство (`auto`/`cuda`/`cpu`), квантизация;
- `language` — `null` (автоопределение), `ru` или `en`;
- `hotkey` — комбинация push-to-talk, по умолчанию `ctrl+space`;
- `vad` — пороги детектора речи (Silero VAD);
- `chunking.max_chunk_s` — записи длиннее 30 с режутся на куски по паузам
  и распознаются последовательно;
- `insertion.method` — `paste` (буфер обмена + Ctrl+V, содержимое буфера
  восстанавливается) или `type` (посимвольный ввод);
- `cloud` — облачный режим (ниже).

Путь к конфигу передаётся через `--config path\to\config.yaml`; без него
используются значения по умолчанию.

## Облачный режим (опционально)

По умолчанию выключен — приложение полностью офлайн. Чтобы распознавать через
OpenAI-совместимый Whisper API (например, агрегатор polza.ai):

```yaml
cloud:
  enabled: true
  base_url: https://api.polza.ai/v1
  api_key_env: POLZA_API_KEY
  model: whisper-1
```

и задайте ключ в переменной окружения:

```powershell
$env:POLZA_API_KEY = "sk-..."
```

Ключ в конфиге и коде не хранится. Если ключ не задан или сеть недоступна,
приложение сообщает об этом явно и не падает молча.

## Как решаются известные проблемы Whisper

- **Галлюцинации на тишине**: аудио уходит в ASR только если Silero VAD
  подтвердил речь; сегменты с низкой уверенностью отбрасываются; известные
  фразы-артефакты («Субтитры сделал…», «Thanks for watching» и т. п.)
  вырезаются постфильтром.
- **Холодный старт**: модель грузится один раз при запуске в фоновом потоке,
  горячая клавиша активируется только после загрузки.
- **Длинная диктовка**: записи длиннее 30 с режутся по паузам VAD на куски,
  распознаются последовательно и склеиваются.
- **Латентность**: время от отпускания клавиши до вставки текста пишется в лог
  (`Latency: …`). Цель: < 2 с для 10-секундной фразы на GPU, < 5 с на CPU.

## Тесты

```powershell
python -m unittest discover -s tests
```

32 теста: нарезка на чанки, фильтр галлюцинаций, конфиг, пайплайн с
подставным ASR, облачный клиент против локального мок-сервера.
Тестовые файлы: `testdata\silence_5s.wav` (тишина — проверка, что ASR не
вызывается) и `testdata\jfk_en.wav` (11 с английской речи, public domain).
