## 2. Telegram-бот (websites/RedRoomBot/redroom-bot/)

**Стек:** Node 20 + TypeScript + grammY, SQLite (better-sqlite3), YCLIENTS REST API v1. Бот: `@rrsmanage_bot`.

### Функции (полностью реализованы и проверены)
- **Уведомления о записи** — создание / перенос / отмена, отдельно клиенту, сотруднику (если привязал телефон) и админу. Перенос детектит именно то, что изменилось (время/мастер/услуга), а не гадает.
- **Двойное напоминание** — за 3 дня и за 1 день до занятия (`REMINDER_OFFSETS_HOURS` в `.env`), клиенту и мастеру.
- **Мост телефон↔Telegram** — жест «Поделиться номером» связывает и клиентов, и сотрудников (сверка со списком сотрудников YCLIENTS).
- **Лид-трекинг:** при получении контакта — админу алерт «Поделились контактом»; если человек дальше ничего не делает в боте 3 минуты — второй алерт «Нужно связаться самому» (deep-link прямо в чат). Сотрудники исключены из этой логики.
- **Триал** «первое занятие бесплатно» за подписку на @redroomstudio (бот проверяет через `getChatMember`).
- **Меню:** услуги/цены, мои записи, связаться с администратором.

### Что нужно проверить перед боевым запуском
- **Бот должен быть админом канала @redroomstudio** — не проверено вживую, без этого триал-флоу всегда будет отвечать общей ошибкой вместо «не подписан».
- Живой end-to-end тест (создать/перенести/отменить настоящую тестовую запись в YCLIENTS) — сознательно не делался, чтобы не трогать боевую систему без разрешения.
- Цены репточки/звукозаписи в `src/content.ts` — TODO сверить.
- Сейчас бот работает **локально** (dev-режим, `tsx watch`), не как сервис/демон — для продакшена нужен хостинг (см. раздел про CRM ниже — вероятно, стоит разместить рядом).

### Полная история находок и фиксов — в памяти проекта
Технические детали (архитектура детекта событий, найденные и исправленные баги, инциденты) подробно записаны в auto-memory ассистента, не дублируются здесь ради краткости. Коротко: был найден и исправлен серьёзный баг (ложные уведомления о «новой записи» для броней, сделанных заранее), баг с дублирующимися процессами бота, гонки при параллельном опросе — всё исправлено и проверено на реальных данных.

## 3. CRM (отдельный репозиторий `redroomstudio-crm`)

Делает товарищ владельца, отдельным AI-агентом. Изучена через технический отчёт (`docs/INTEGRATION-REPORT.md`), сгенерированный по промпту.

**Что это:** локальный (не облачный) инструмент на машине владельца — Node без фреймворков, `node:sqlite`, синкает YCLIENTS почасово в SQLite, дашборд — статический HTML. Однопользовательский, не SaaS.

**Точка интеграции с ботом — уже готова:** таблица `lead` со специально предусмотренным каналом `'Telegram'`. Рекомендация авторов CRM: `source='TELEGRAM', external_id=tg_user_id`. Эндпоинт `POST /api/leads`.

**Блокеры для стыковки прямо сейчас:**
1. API CRM без аутентификации, слушает только `localhost:8787` — нет доступа снаружи.
2. Неизвестно, будут ли бот и CRM на одной машине/VPS в проде.
3. Вебхуков нет ни в одну сторону — интеграция может быть только push (бот → CRM).
4. Создание реальной записи (брони) в YCLIENTS не реализовано ни в боте, ни в CRM — общая незакрытая задача, решать один раз.

**Решение (2026-07-10):** интеграцию откладываем до стабилизации инфраструктуры (одна машина/VPS для бота и CRM + аутентификация API CRM). Когда решится — подключение маленькое: один `POST /api/leads` рядом с уже готовым алертом в bot.ts.

### Шаг 1 — Домен: РЕШЕНО ✅
